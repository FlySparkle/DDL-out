# ADR 004：响应式边栏模式

## 状态

于 2026-07-13 采纳。

## 背景

原有“自适应桌面侧边栏”只在桌面平台出现，并将固定、悬浮和响应式行为绑定在
一个布尔值中。用户无法明确选择导航方式，窄窗口下也缺少统一的降级规则。

## 决策

设置提供不区分平台的两种边栏模式：

- 浮动和固定边栏共享同一组 Material 3 导航按钮。按钮高 56px、圆角 16px，
  展开时显示图标与文字，导航按钮之间保留 8px 间距。
- 浮动边栏通过应用栏按钮打开；在 Android 和 iOS 上，屏幕任意横向位置开始的
  右滑也可以打开边栏。
- 固定边栏支持手动收展；收起时悬停会以覆盖方式展开，不挤压右侧内容。

固定边栏的手动收展按钮位于左下角，并复用导航按钮的尺寸、圆角、内边距和文字
样式。边栏右侧统一使用 16px 圆角；收展时文字随宽度进行位移和透明度过渡。
鼠标进入 450ms 后展开，离开 500ms 后收起；延迟任务在鼠标重新进入、离开或
用户手动操作时取消，避免快速划过边栏导致闪动。

固定模式按可用宽度响应：

- 小于 800px 时降级为浮动边栏。
- 800px 至 1099px 时默认收起。
- 1100px 及以上时默认展开。

用户手动选择的收展状态在当前固定边栏实例中优先于自动默认状态。窗口窄到固定
模式不可用时，页面切换为浮动边栏；重新具备固定空间后重新按宽度选择默认状态。

设置使用 `navigation_mode` 持久化。旧版 `adaptive_desktop_sidebar=true` 迁移为
固定模式，其余情况迁移为浮动模式。

## 影响

- 移动端、桌面端和不同窗口宽度共享同一套设置与导航语义。
- 两种模式使用同一份主页和设置目的地定义、同一个按钮组件及全局字体。
- 路由使用 `ShellRoute` 承载应用级导航。固定模式只替换 Shell 右侧的路由内容，
  因此边栏及其手动收展状态在主页和设置页之间持续存在。
- 浮动模式不在 Shell 中渲染边栏；每个全屏页面持有自己的 Drawer。从主页进入
  设置时压入路由历史，因此系统返回会回到主页而不是退出应用。
- Shell 负责决定固定模式是否可用，边栏组件负责固定模式内部的收展与悬停动画。
